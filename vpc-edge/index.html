ESun VPC Edge Features

1. OpenVPN SSL remote user VPN

2. IPSec remote user VPN

3. Okta/SAML single sign on(SSO) for both OpenVPN and IPSec

4. Cloud-to-Cloud IPSec Tunnel

5. Token-expiration secures the remote user VPN link

Access the ESun VPC Edge EC2 Instance with HTTP

After a new ESun VPC Edge is launched, http://public_ip_of_ec2_instance can access the configuration page. http://public_ip_of_ec2_instance/login is for the admin to login. The default username and password to login a new instance:

username: admin

password: admin123

Access the ESun VPC Edge EC2 Instance with HTTPS

HTTPS is recommended to access the EC2 instance. And it is required by Okta/SAML Single Sign On(SSO). To make HTTPS work, it needs to upload the certs. The "Settings" section has the "Upload Certs for Web Service" to upload the certs.

IMPORTANT: Once the certs are applied, then the http will be closed. And it requires to to re-type the URL to broswer:

https://public_ip_of_ec2_instance/login

Best Security Pratice of Web Login

The ESun VPC Edge has the Web Login to administrate the server, and it has the SMTP server settings to send the e-mail.

Important:

1. Please change the default password of the web portal after the 1st time login. The default username/password is admin/admin123 .

2. Please set the "Email for Forgot Password" to get the password back if the password is forgotten.

If it doesn't configure an email SMTP server, it will use a local default email server, which will show the "From" no-reply@voipeye.com in the receiver's e-mail box . But, please set "SMTP to Send Email" to a well known SMTP server(e.g. your comapny email server, or gmail), or the email to the endpoint users or admin may go the spam folder of the email receivers. If it is gmail, then password is the App password, not the gmail login password.

The password of the Web Login is hashed before it is saved into the server database. That means the server doesn't store the plain text of the Web portal password.

The password to login the SMTP server is encrypted before it is stored into the server database. The keys to encrypt passwords are unique for each new launched ec2 instance.
