Start IPSec Remote User VPN

After a new AMI is launched, please go to "Services Status" to turn on IPSec and turn off OpenVPN.

Please go to "Configure Endpoint Routes" of IPSec or SAML to configure the routes for traffic, or it will reject the client's connection.

Start OpenVPN SSL Remote User VPN

After a new AMI is launched, please go to "Services Status" to turn on OpenVPN and turn off IPSec.

Please go to "Configure Endpoint Routes" of OpenVPN or SAML to configure the routes for traffic, or it may reject the client's connection.

IPSec Remote User VPN or OpenVPN VPN

Both IPSec remote user VPN and OpenVPN can connect the remote worker to the virtual private cloud. By default, both IPSec and OpenVPn are turned on. However, if it goes to production, then it should run on one mode only in one EC2 instance and turn off other modes. Or it may cause the traffic routes conflicting issues.

For the iPhone/Android, IPSec remote user VPN should be the better choice. It does not require to install any extra client software, and less firewall issues.

If the endpoint user with a laptop is traveling around in different locations, then OpenVPN is the better choice, as it will hit less firewall blocking issues than IPSec. But it requires to install the OpenVPN client.
